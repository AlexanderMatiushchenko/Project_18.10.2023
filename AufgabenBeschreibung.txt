1.Schwierigkeitsstufe: Fügen Sie ein Anmeldeformular über E-Mail und Passwort hinzu. Wenn ein solcher Benutzer in localStorage vorhanden ist, authentifizieren Sie den Benutzer und zeigen Sie eine Benachrichtigung über den erfolgreichen Login an. Wenn die Daten fehlerhaft sind, zeigen Sie eine Benachrichtigung an, dass der Login fehlgeschlagen ist. Die Authentifizierung kann auf zwei Arten implementiert werden: 1) über localStorage, in diesem Fall müssen Sie auch die Registrierung implementieren; 2) über jsonplaceholder über die API /users. Wenn der Benutzer authentifiziert ist, erscheint für ihn ein Formular mit 3 Eingabefeldern: Adresse, Preis und Kosten, die der Benutzer ausfüllt und per Post-Methode sendet (verwenden Sie jsonplaceholder). Wenn die Antwort mit dem Status 201 zurückkommt, zeigen Sie eine Benachrichtigung an, dass die Bestellung erstellt wurde.
Unterhalb des Formulars befindet sich ein Container mit vier Schaltflächen: Bezahlen, Senden, Akzeptieren, Bestellung abschließen. Die Schaltflächen sollten erst erscheinen, nachdem die Bestellung erstellt wurde. Bei Klicks auf die Schaltflächen Bezahlen, Senden, Akzeptieren, Bestellung abschließen sollten entsprechende Benachrichtigungen angezeigt werden. Zum Beispiel, beim Klicken auf Bezahlen – "Bestellung bezahlt", beim Klicken auf Senden – "Bestellung versendet, bitte warten".

Es gibt auch eine Schaltfläche "Ausloggen", bei deren Betätigung das Benutzeroberfläche für die Bestellungserstellung und die Statusschaltflächen verschwinden.

*Zusätzliche Aufgabe: Bei Klicks auf die Schaltflächen Bezahlen, Senden, Akzeptieren, Bestellung abschließen einen fetch GET-Request an /posts (jsonplaceholder) senden und in der Benachrichtigung neben der Anzeige der Statusmeldungen auch Daten vom Server anzeigen, wobei während der Verarbeitung des Requests ein Ladezustand angezeigt wird (Skeleton usw.).
